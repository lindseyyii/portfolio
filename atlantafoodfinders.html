Atlanta
Food Finders

A collaborative web application project using Django and Google Maps API that helps users find restaraunts in Atlanta

Watch our demo

User Stories

User Authentication is important because it ensures that each user has their own personalized account, allowing them to save favorites or keep track of how they have used the application.

Restaurant Search gives users flexibility in how they find restaurants, optimizing options for the user.

Geolocation utilizes Google Maps to show precise locations to users, allowing them to easily view which restaurants are near them.

Restaurant Details is important because it allows users to get a more in-depth view of restaurants they may be interested in and make more informed decisions.

Favorites is important because it allows users to easily keep track of their favorite restaurants and access their details quickly.

This is important because both extra goals add an extra layer of interactivity and convenience for the user, incentivizing them to continue using the platform.

The Process

1. Planning Phase: Outlining the Project Goals

This project consisted of two sprints (1st sprint: user story A-C; 2nd sprint: user story D-F). For initial organization, the team created a shared Google Calendar, Trello board, and Discord server to ensure efficient communication. For each sprint, the team held a meeting to discuss incomplete user stories. We created a "Daily Updates" Discord channel and used tags/delegate work cards in Trello to keep each other up to date.
→
2. Design Phase: Creating Mockups and Prototypes

We used Figma to create a basic user flow and wireframes for the UI. We then created a basic brand identity and applied it to the wireframes to create prototypes. The basic decisions regarding the branding and setup were made with the application's intentions and functionality in mind. We wanted it to be clean and user-friendly, so we kept the colors and user flow simple to avoid overwhelming users.
→
3. Development Phase: Building the Project

Our team initially struggled with the learning curve regarding Git and GitHub, but we were able to learn and adapt quickly. We used the framework of our Django tutorial to initialize our project. We broke up each user story into smaller tasks and made sure to have other team members review our code before merging anything to the repository. This gave members a chance to share any challenges they were facing and helped us stay up to date with our progress.
→
4. Testing & Deployment: Ensuring Functionality

We made sure to use Git and branches effectively to test our work before merging it into main to avoid any conflicts. The process of testing was extensive and occurred throughout the creation of the project, rather than solely at the end.
