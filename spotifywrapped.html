Spotify
Wrapped

A collaborative web application project using Spotify's API that allows users to see their music wrapped.

Watch our demo

User Stories

User Authentication is important because it ensures that each user has their own personalized account, allowing them to save favorites or keep track of how they have used the application.

Restaurant Search gives users flexibility in how they find restaurants, optimizing options for the user.

Geolocation utilizes Google Maps to show precise locations to users, allowing them to easily view which restaurants are near them.

Restaurant Details is important because it allows users to get a more in-depth view of restaurants they may be interested in and make more informed decisions.

Favorites is important because it allows users to easily keep track of their favorite restaurants and access their details quickly.

This is important because both extra goals add an extra layer of interactivity and convenience for the user, incentivizing them to continue using the platform.

The Process

1. Planning Phase: Outlining the Project Goals

This project was split into two sprints (1st sprint: user authentication, implementing Spotify's API, front-end designs; 2nd sprint: connecting front-end and back-end, profile management system, responsive UI across various resolution sizes). We used tools like Google Calendar, Trello, and Discord for team coordination. Each sprint began with a planning meeting, and we would stay accountable and troubleshoot problems as a team with semi-weekly updates Discord channel.
→
2. Design Phase: Creating Mockups and Prototypes

We used Figma to create detailed wireframes and user flows, followed by prototypes that reflected the Spotify's branding and functionality. The front-end developers focused on making the design visually appealing while ensuring it worked across different screen sizes. Bright colors and abstract shapes were incorporated to give a modern and electric feel while enhance visual engagement.
→
3. Development Phase: Building the Project

The main challenge came with user authentication, as we had to integrate Spotify’s API for unique user profiles. We overcame this by working collaboratively and refining our user authentication flow. Each user story was broken into smaller tasks, and code reviews were essential for maintaining quality and ensuring progress. Collaborating and supporting each other helped keep the project on track.
→
4. Testing & Deployment: Ensuring Functionality

Throughout the project, we ensured testing was continuous, using Git branches for testing new features and avoiding conflicts. Testing was done iteratively, allowing us to catch and fix bugs early, which was critical for smooth deployment.
